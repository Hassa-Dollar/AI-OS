**This directory = the system's _decisions and truth_** — ADRs, contracts, invariants, glossary (the *why*).
For *how to operate* the system see `docs/`; for cross-cutting *wisdom* (patterns, postmortems, research) see
`knowledge/`.

## What this repo is
**AI‑OS** is the operating system (this scaffold). The **products it builds** live as self‑contained,
**extractable components** under `components/<name>/`, each governed by exactly one **profile**
(`profiles/<family>/<variant>/`) and coupled to other components **only** through a **contract** in
`architecture/contracts/` — never by importing each other's source (one‑way isolation, ADR‑0002). The OS
reaches into components; components never reach out. When a product is active, its full plan sits at the repo
root (e.g. `shrink-build-plan.md`).

## The OS engine (durable — change only via an ADR)
| Area | Path | Responsibility |
|---|---|---|
| Determinism layer | `scripts/` | dispatch · gate · ship · land · memory (`db.sh`) · ledger — the mechanical pipeline (P10) |
| Decisions & truth | `architecture/` | ADRs (`adr/`) · contracts (`contracts/`) · invariants · glossary |
| Roles & prompts | `agents/`, `prompts/` | role cards + the prompts injected into each worker run |
| Profiles | `profiles/` | reusable stack templates: conventions · CI env · skeleton · role→model bindings |
| CI | `.github/workflows/` | `os-ci` (universal) + `product-ci` (per‑component matrix) — ADR‑0006/0013 |
| Memory & ledger | `reports/`, `knowledge/`, `reports/metrics/memory.db` | dated event/knowledge/bug memory + the append‑only ledger (ADR‑0016) |
| Work queue | `tasks/` | task specs as files: `tasks/active/*.md` (the memory DB does **not** hold tasks) |

## Per‑component internals
Each component documents its own build/run/architecture in `components/<name>/README.md` + `conventions.md`,
so it stays extractable. This page does **not** duplicate them.

## Invariants & decisions (do not contradict)
- Invariants: [`invariants.md`](./invariants.md) — always hold; changing one needs an ADR.
- Decisions: the ADR index in [`adr/`](./adr/) — start at `0001`.

## How this page stays current
The inventory block is generated by `scripts/handoff.sh` (run after every merge by `land.sh`, or manually);
a planned CI check fails the build if it's stale. Everything else here is durable — it describes the OS
engine and the component/profile/contract *pattern*, not any one product.
